# ISO CyberSecurity Management System

## Описание проекта

`ISO CyberSecurity Management System` — учебная full-stack система менеджмента информационной безопасности для ООО «MediNova Clinic». Проект демонстрирует работу с идеями ISO 27001 и ISO 27005: описание организации, реестр активов, анализ угроз и уязвимостей, качественная и количественная оценка рисков, планирование контролей, политики безопасности и итоговый отчет.

Проект не является инструментом реальной сертификации и используется только как образовательная симуляция.

## Цель проекта

Показать полный цикл построения ISMS для медицинской организации: определить область применения, классифицировать активы, оценить риски, предложить меры контроля и подготовить материалы для защиты университетской работы.

## Используемые технологии

- Frontend: React, Vite, TypeScript, React Router, Bootstrap 5, React-Bootstrap, Recharts, Axios, jsPDF, html2canvas.
- Backend: Node.js, Express, SQLite, better-sqlite3.
- Testing: Vitest.
- Export: Markdown, CSV, PDF на стороне клиента.

- `client` — React/Vite приложение.
- `server` — Express API и SQLite-логика.
- `database` — файл базы данных `isms.sqlite`.
- `docs` — учебные политики и справочные материалы.

```bash
npm run seed
```

Команда пересоздает учебные данные: организацию, подразделения, активы, угрозы, риски, контроли, политики и вопросы checklist ISO 27001.

1. Выполнить `npm run seed`.
2. Запустить сервер: `npm run start:server`.
3. Запустить клиент: `npm run dev:client`.
4. Открыть `http://127.0.0.1:5173`.

## Формулы риска

### ISO 27005 qualitative model

Используется учебная формула:

```text
risk = round((((TP - 1) + (VL - 1) + (SEV - 1) + (5 - DET)) / 16) × 100)
```

- `TP` — вероятность угрозы.
- `VL` — уровень уязвимости.
- `SEV` — серьезность / влияние на бизнес.
- `DET` — возможность обнаружения. Чем выше DET, тем ниже риск.

Уровни:

- риск меньше 20% — низкий;
- 20–40% — средний;
- больше 40% — высокий.

### Quantitative model

```text
SLE = Asset Value × Exposure Factor
ALE = SLE × ARO
```

`SLE` показывает ожидаемый ущерб от одного события, `ALE` — ожидаемый годовой ущерб.

## ISO 27001

В проекте ISO 27001 представлен как образовательный checklist из 38 вопросов по разделам: контекст организации, лидерство, планирование, поддержка, операционная деятельность, оценка эффективности, улучшение и контроли Annex A.

## ISO 27005

ISO 27005 представлен как методика управления рисками: идентификация активов, угроз и уязвимостей, расчет риска, выбор мер обработки риска и оценка остаточного риска.

## Ограничения

- Это учебная симуляция, а не официальный инструмент сертификации.
- Тексты стандартов не копируются дословно.
- Для реального внедрения требуются аудит, доказательства внедрения, юридическая проверка и регулярный мониторинг эффективности контролей.
